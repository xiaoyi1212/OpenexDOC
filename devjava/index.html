    OpenEX development for Java

    • 本文介绍了如何用Java语言对OpenEX进行扩展开发
    • 开发之前您需要了解OpenEX每个包的作用是什么

    入口

    ex.openex

    Main

    OpenEX的程序入口点在ex.openex.Main类中，在这个类中我们首先观察类中两个常量

    ex.openex.Main
    public static final String compile_version = "OpenEX_ScriptCompile_AbstractSyntaxTree_v0.1.3";
public static final String runtime_version = "OpenEX_ScriptRuntime_AbstractSyntaxTree_v0.1.2";
    OpenEX版本号遵循OpenEX_<模块>_<版本>_v<版本号>的格式
    compile_version代表的是编译器的版本
    runtime_version代表的是解释器的版本
    如果你要修改OpenEX的代码并发布请更改版本号,防止与正式版的OpenEX版本冲突

    CommandManager

    OpenEX启动后会将传入的命令行交由ex.openex.CommandManager进行处理
    方便后续的代码维护以及功能管理

    ex.openex.CommandManager
    static ArrayList<String> filename = new ArrayList<>();//输入的文件名
static boolean iscompiler = false,isbuffer = false;//是否开启编译缓存
public static void command(String[] args);//命令处理
public static boolean isCompiler();
public static boolean isBuffer();

    编译器ex.openex.compile

    LexicalAnalysis

    该类负责OpenEX的初步词法分析,架构参考《两种自制脚本语言》的电子书 解析输入的源代码文件并输出Token词素们

    ex.openex.compile.LexicalAnalysis
    public static final int INTEGER = 0, //OpenEX的Token标记
            DOUBLE = 1,
            SEM = 2,
            STRING = 3,
            KEY = 4,
            NAME = 5,
            TEXT = 6,
            LP = 7,
            LR = 8,
            LINE = 9,
            END = 10;
private Token lex() throws NullPointerException; //每次解析流程取一次词素,如果发生空指针异常代表解析完成
public static class Token {
    public Token(int type, String data, int line);
    public int getLine(); //为编译器精准报错用，代表该词素所在的文件行
    public int getType() ; //词素标记
    public String getData(); //词素的内容
    @Override
    public String toString();
}

    parser

    该包负责OpenEX的语法解析,解析出对应的每条语句，并生成对应的语法树
    OpenEX编译器的架构比较独特,其语法树与常规的编译器语法树结构不同
    OpenEX的语法树每个子节点都具备对该节点语句的语法/语义处理能力

    ex.openex.compile.parser.Parser
    public BaseParser getParser(Compiler c) throws EOFException; //一级语法分离器,分离出一些调用，定义以及块语句
public static class SubParser; //子解析器,负责分离出块语句内部的语句，遇到多级块嵌套时，采用递归调用处理
    ex.openex.compile.parser.BaseParser
    public ByteCode eval(Parser parser, Compiler compiler) throws CompileException; //子节点的语法、语义处理器,直接翻译成虚拟机字节码输出

    expression

    该包负责OpenEX的算术表达式和布尔表达式的解析,解析器直接输出对应的字节码

    • ExpressionParser -表达式分析器，判断表达式类型,决定采用哪种解析器翻译表达式
    • ArithmeticExpression - 算术表达式解析器
    • BoolExpression - 布尔表达式解析器
    • SyntheticExpressions - 综合表达式解析器,负责翻译布尔表达式与算术表达式混合的表达式
    警告!

    OpenEX-AST2.0版本以内对综合表达式的解析非常欠佳
    解析BUG巨多不说,运算符优先级也很混乱

    解释器ex.openex.astvm

    thread

    该包包含了对OpenEX的线程管控,其底层实现还是依靠java.base\java.lang.Thread 也负责一些全局变量以及各种脚本定义过的函数的存储

    • ThreadManager -负责线程管控,用于启动, 中断, 释放某个线程
    • ThreadTask -OpenEX的线程存储单元

    obj

    该包内代表了OpenEX内所有变量在Java代码中的表示

    • ExObject -类型基类,任何变量类型都要继承此类
    • ExValue -普通变量类,存储一些普通的变量
    • ExValueName -变量名,OpenEX虚拟机原因,实际操作变量时都是由该类顶替实际变量
    • ExArray -数组变量,继承ExValue,代表OpenEX的数组

    exe

    该包内是OpenEX内的各种执行器

    • Executor -执行引擎,操作栈的使用以及库调用,线程管理都要通过Executor类映射
    • LibraryLoader -库加载器,由include语句决定实际导入的库
    • Script -代表一个编译后的脚本文件

    code

    该包内是OpenEX的所有解释器字节码
    实际上每个字节码会内置一个子执行引擎负责具体的解释执行
    但一些控制操作栈等功能还要靠Executor去映射

    • ex.openex.astvm.exe.code.struct -代表一些流程语句的字节码
    • ex.openex.astvm.exe.code.opcode -代表运算字节码
    • ByteCode -所有字节码都要实现ByteCode接口

    lib

    该包内是OpenEX本地库的java实现 所有OpenEX的本地库都在该包内定义

    • RuntimeLibrary -所有OpenEX本地库都继承此接口
    • RuntimeLibrary.RuntimeFunction -每个OpenEX函数的具体实现要继承此接口
    • Function -该类不算做本地函数,是OpenEX脚本中定义的函数在Java代码中的表示

    异常ex.openex.exception

    OpenEX异常代表了编译中发生的无法解析语句错误，解释器运行时异常等

    • CompileException -编译型异常
    • VMRuntimeException -解释器运行时异常
    • ReturnException -特殊的异常
    ReturnException
    • 准确来说该异常并不一定代表用户代码有问题
    • 它是return语句的异常中断实现
    • 如果用户代码位置正确，解释器会正确处理ReturnException
    • 如果用户定义在函数块外部，会造成虚拟机底层发生ReturnException并终止程序

    如何添加您的代码

    构建

    本教程建议您使用IDEA2022以上版本构建此项目,将项目导入到您的IDEA中，直接点击右上角的运行即可调试您的代码,也可以通过构建JAR包来发布您修改过的OpenEX版本

    其他IDE的用户中 使用VisualStudioCode的用户请使用以下代码构建项目

    编译源代码
    javac -d . src\*.java

    运行源代码
    java -Dfile.encoding=UTF-8 -Dsun.stdout.encoding=UTF-8 -Dsun.stderr.encoding=UTF-8 -classpath %项目根目录路径% ex.openex.Main

    添加代码

    根据需求不同,您可以更改OpenEX的编译器或解释器中代码的实现来完成您自己的功能

    注意!
    • 修改后的OpenEX便不在代表OpenEX主要版本
    • 您需要将ex.openex.Main中的两个常量更改为您自己的版本号以区分OpenEX
